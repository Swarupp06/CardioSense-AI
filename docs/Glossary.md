# Purpose

This glossary contains medical, machine learning, software engineering, and Git terminology used throughout the CardioSense AI project.

The objective is to provide clear explanations of technical concepts to improve understanding, maintain consistency across project documentation, and support future learning.

---

# Medical Terms

| Term | Definition |
|------|------------|
| Angina | Chest pain caused by reduced blood flow to the heart muscle. |
| Cholesterol | A fatty substance found in the blood. High cholesterol can contribute to plaque buildup in arteries and increase the risk of cardiovascular disease. |
| ECG (Electrocardiogram) | A medical test that records the electrical activity of the heart to detect abnormalities. |
| Blood Pressure | The force exerted by circulating blood against the walls of blood vessels. High blood pressure is a major risk factor for heart disease. |
| Heart Disease | A general term describing disorders that affect the heart, including coronary artery disease, heart attacks, and heart failure. |

# Machine Learning Terms

| Term | Definition |
|------|------------|
| Machine Learning | A branch of Artificial Intelligence where algorithms learn patterns from data to make predictions or decisions. |
| Dataset | A structured collection of data used for training and evaluating machine learning models. |
| Feature | An input variable used by a machine learning model for prediction. |
| Target Variable | The output variable that the model attempts to predict. |
| Binary Classification | A machine learning task where the model predicts one of two possible classes. |

# Software Engineering Terms

| Term | Definition |
|------|------------|
| SRS | Software Requirements Specification. A document describing the software's functional and non-functional requirements. |
| Architecture | The high-level design describing how different components of the software interact. |
| Module | A self-contained component responsible for a specific functionality within the system. |

# Git & GitHub Terms

| Term | Definition |
|------|------------|
| Git | A distributed version control system used to track changes in source code. |
| GitHub | A cloud platform used to host Git repositories and collaborate on software projects. |
| Repository | A project folder managed by Git that contains source code, documentation, and version history. |
| Commit | A snapshot of the project's current state saved in Git history. |
| Branch | An independent line of development used to work on features without affecting the main codebase. |
